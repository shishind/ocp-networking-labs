# Contributing to OCP Networking Labs

Thank you for your interest in contributing to the OCP Networking Labs project!

## How to Contribute

### Reporting Issues

If you find errors, typos, or have suggestions for improvements:

1. Check if the issue already exists in the GitHub Issues
2. If not, create a new issue with:
   - Clear title describing the problem
   - Which lab/week it affects
   - What you expected vs. what you found
   - Suggested fix (if you have one)

### Suggesting New Content

We welcome suggestions for:
- Additional labs or exercises
- New troubleshooting scenarios
- Updated commands for newer OCP versions
- Additional cheat sheets

### Submitting Changes

1. Fork the repository
2. Create a new branch (`git checkout -b feature/your-feature-name`)
3. Make your changes
4. Test your changes (run the commands if possible)
5. Commit with clear message (`git commit -m "Add: description of change"`)
6. Push to your fork (`git push origin feature/your-feature-name`)
7. Open a Pull Request

### Lab Structure Guidelines

When adding or modifying labs, please maintain the consistent structure:

1. **Learning Objectives** - What students will learn
2. **Plain English Explanation** - Concepts without jargon
3. **Hands-On Lab** - 5-6 practical exercises
4. **Self-Check Questions** - With answers
5. **Today I Learned (TIL)** - Template for notes
6. **Commands Cheat Sheet** - Quick reference
7. **What's Next** - Preview of next topic

### Code of Conduct

- Be respectful and inclusive
- Focus on helping others learn
- Provide constructive feedback
- Keep discussions professional

## Questions?

Open an issue with the "question" label and we'll help!

Thank you for helping make this resource better for everyone!
